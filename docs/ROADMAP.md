# Roadmap：新手體驗補完（解放條件資料層＋引導模式）

> 狀態：**已完成**（2026-07-07 提前開工，階段 0~3 全數完成）。
> 產出：解放條件資料層（`unlocks.json`＋匯入/驗證腳本）、搜尋引擎進度篩選、新手引導模式（`/guide`）、[DATA-COVERAGE.md](DATA-COVERAGE.md)。
> 後續擴充見 README「擴充方向」與 DATA-COVERAGE 待補清單。以下保留原始計畫供參照。

**定位一句話**：「不是又一個畢業配裝模擬器——是告訴剛入坑的你，現在做得出什麼、接下來該做哪一套的練功路線工具。」市面上唯一「進度導向」配裝工具。

## 背景

這是魔物獵人崛起:破曉的配裝搜尋工具，已具備：68 套（14 武器 × 初心/拓荒/進階/畢業）漸進式 preset、依 rarity 上限的階段限裝、EFR 傷害模型、裝飾珠求解。目前短板：遊戲進度用 rarity 近似，無法精確回答「以玩家目前的任務進度，這件裝備到底能不能做」。

本次目標（依優先序）：

- A. 建立「解放條件」資料層：裝備可製作的精確條件（對應任務進度/MR）
- B. 新增「引導模式」：玩家輸入目前進度 → 精確篩選 + 推薦下一套目標
- C. 全程不破壞現有功能，漸進上線

## 總體規則

1. 每階段動工前先列計畫確認；每階段完成即 commit。
2. 現有搜尋、preset、EFR、珠子求解的行為不得改變；新功能以可開關的方式疊加。
3. 資料的不確定性要標註，沿用本專案既有的「信心度標註」風格，不確定的解放條件寧可標「未驗證」也不要假裝精確。

## 階段 0：資料可行性偵察

1. 調查解放條件資料的可得性，逐一回報：
   - 現有資料源（Kiranico）是否包含防具/武器的解放條件欄位（出現時期、對應任務、MR 需求）？現有匯入腳本能否取得？
   - 若不完整，盤點替代方案：其他資料站、社群 wiki、或退而求其次的粗粒度方案（見下）
2. 提出資料 schema 提案：每件裝備的 unlock 欄位設計，至少要能表達：章節/階段（村/集會所/MR 段落）、關鍵任務、MR 等級需求、資料信心度（confirmed / inferred / unverified）
3. 粗粒度後備方案（若精確任務資料難以取得）：以「遊戲進度里程碑」為單位（例如：上位開放、MR 開放、MR10、MR50、MR100、各 TU 段落），將裝備映射到里程碑。對新手而言，里程碑粒度已足以回答「我現在做不做得出來」，且與現有 rarity 限裝可以並存（精確資料逐步覆蓋近似資料）。
4. 回報偵察結果與建議路線，選定後進入階段 1。

## 階段 1：解放條件資料層

1. 依選定路線建立/擴充資料匯入腳本，產出 unlock 資料（獨立資料檔，不與現有裝備資料混寫，以 id 關聯）。
2. 寫資料驗證腳本：引用完整性、每件裝備至少有一個粒度的 unlock 資訊或明確標註 unverified、統計覆蓋率報告（多少 % 裝備有 confirmed 級資料）。
3. 不動 UI，先讓搜尋引擎支援「以進度篩選」作為 rarity 限裝的精確化替代（可透過旗標切換新舊篩選，對照驗證結果合理性）。
4. commit，回報覆蓋率與抽查結果。

## 階段 2：引導模式（新手的主入口）

1. 新增「我的進度」輸入：極簡設計，新手不查攻略也答得出來——目前在村/集會所第幾星或 MR 多少、（可選）已擊破的關鍵魔物。持久化到 localStorage，沿用專案既有慣例。
2. 引導頁核心輸出：
   - a. 「你現在的最佳配裝」：以進度精確篩選後，套用對應階段的 preset 流派，給出當下可製作的最佳解
   - b. 「下一套目標」：同流派的下一階段 preset，列出差距——還缺哪些裝備、各需要打什麼素材/魔物（素材資料若現有資料源沒有，先列裝備名與所需魔物，細節標註導向 Kiranico 連結）
   - c. 每件推薦裝備標註解放資訊與信心度
3. UI 原則：引導模式是預設著陸體驗的候選，但先做成獨立分頁/入口，不改動現有進階搜尋的動線；文案面向新手，避免術語轟炸（第一次出現的術語給一行白話說明）。
4. commit，回報，附幾個典型新手情境的端到端測試（例：剛進上位的太刀玩家 / MR 20 的弓玩家）。

## 階段 3：收尾

1. README 更新：新增「這個工具為誰而做」段落，寫明進度導向定位、解放資料的來源與覆蓋率、信心度說明。
2. 效能檢查：unlock 資料維持 lazy loading 慣例，不拖慢首屏。
3. 產出 docs/DATA-COVERAGE.md：解放資料覆蓋率現況與待補清單，方便日後社群回報錯誤時逐步修正。
