# 武器派生歸類待辦

> 來源：2026-07-07 Claude 影片初步分析筆記。
> 目標：把《Monster Hunter Rise: Sunbreak》全武器資料整理成可用的派生樹、門檻傳播與引導資訊。

## 背景

- 參考影片：https://www.youtube.com/watch?v=sYVvyKA44oc
- 影片內容是遊戲內加工屋武器派生圖實錄，日文介面，長度約 2 分 18 秒。
- 影片大約每種武器停留 10 秒並快速捲動派生樹。
- 影片能協助確認派生樹的順序、分段、節點數與分岔形狀。
- 影片不適合完整抽取每個節點的武器名稱，因為游標只偶爾停留，無法涵蓋全部武器。

## 可行性結論

可以做派生歸類。

主要資料來源應以既有武器列表資料為主，影片作為抽驗和校正分岔的輔助來源。初步驗證顯示，Kiranico 的武器列表順序本身接近派生樹的連續分段，例如大劍列表中村莊守衛樹、神火樹、爵銀龍樹、雪鬼獸樹、泡狐龍樹等都呈現連續排列；派生樹邊界常可由 rarity 從 R10 重置回 R1-R3 等特徵辨識。

## 實作階段

### 1. 資料分段

- [x] 從現有武器列表資料切出每種武器類型的派生樹分段。（scripts/derive-weapon-trees.mjs，大劍+雙劍先行）
- [x] 使用 rarity 重置作為主要斷點訊號。（實測：rarity 重置一律斷樹，比「重置且可生產才斷」安全——升級限定家族段的父節點在其他段落）
- [x] 使用系材來源作為輔助斷點訊號。（改用素材快照的生產/強化欄位；f/u 皆空=初始配給視同可生產）
- [x] 產出每把武器的 `treeId`、`treeName`、`nodeIndex` 欄位。（src/data/weaponTrees.json，大劍 55 樹+雙劍 56 樹）
- [x] 對大劍先做一輪完整樣本驗證。
- [x] 將同一套規則擴展到 14 種武器。（757 樹／3953 把全覆蓋；74 筆傳播，其中 19 筆蓋 confirmed 者全為金/銀火龍武器——與金銀龍對劍、輝劍火龍同型，二者皆經遊戲內驗證）

### 2. 生產錨點與門檻傳播

- [x] 判斷每把武器是否可直接生產。（素材快照 f 非空，或 f/u 皆空＝初始配給）
- [x] 將可直接生產武器視為傳播重置點。
- [x] 對升級限定武器沿同樹順序繼承前一階的解鎖門檻。（src="tree-propagation"，大劍+雙劍共 12 筆）
- [x] 保留 `confirmed`、`inferred`、`unverified` 等可信度狀態。（傳播值信心度上限 inferred，鏈結構本身是推導）
- [x] 以金銀龍雙劍鏈作為測試案例：演算法獨立重現人工覆寫值（雙炎劍 MR50），並抓到大劍版同型漏網案例（輝劍火龍 mr10→50，前一階火龍吼劍同樣需 A4 傀異化之爪）。
- [x] 補上分岔鏈的特殊處理，避免分支根錯誤繼承前一分支尾端門檻。（升級限定分支根自成一樹、不繼承＝保守低估；39 筆列入可疑清單待影片抽驗）

### 3. 影片抽驗與分岔校正

- [ ] 從影片擷取關鍵截圖，約每種武器 3-5 張，總計約 50 張。
- [ ] 用截圖確認每種武器的大樹分段與分岔形狀。
- [ ] 優先抽驗 rarity 重置不明顯、同魔物多分支、或資料規則判斷不穩的樹。
- [ ] 建立人工校正表，記錄需要覆寫的樹邊界或分岔繼承規則。
- [ ] 將校正結果回寫到資料產生流程，而不是只修改輸出 JSON。

### 4. UI 與引導模式

- [ ] 在武器資料中顯示所屬派生樹。
- [ ] 在引導模式中顯示「這把武器的下一階」。
- [ ] 顯示下一階的解鎖門檻、可信度與資料來源。
- [ ] 對分岔武器顯示可選分支，而不是只顯示單一路徑。

## 風險與處理方式

- 分岔處可能誤判繼承來源。
  - 處理：以影片截圖與人工校正表修正高風險分支。
- 影片無法提供完整武器名稱。
  - 處理：名稱仍以資料來源為準，影片只驗證結構。
- 日文樹名與中文魔物名需要對照。
  - 處理：沿用現有魔物名稱對照或補一份 tree name override。
- rarity 重置不是所有邊界都可靠。
  - 處理：結合系列名、素材來源與可直接生產資訊判斷。

## 驗收條件

- [ ] 每把武器都有派生樹標籤。
- [ ] 同一棵樹內的節點順序可穩定重建。
- [ ] 可直接生產武器能正確重置門檻傳播。
- [ ] 升級限定武器能繼承合理的前置解鎖門檻。
- [ ] 分岔樹有人工抽驗紀錄。
- [ ] 引導模式能顯示目前武器的下一階與分支。

## 建議優先順序

1. 先用大劍建立分段與門檻傳播原型。
2. 擴展到 14 種武器並產出自動檢查報告。
3. 用影片抽驗可疑分岔。
4. 接入 UI 引導模式。
